# ระบบเงินเดือน PTC (Payroll & Slip System)

เว็บแอปคำนวณเงินเดือนและออกสลิปเงินเดือนรายบุคคล (Export PDF) สร้างจากไฟล์ต้นฉบับ
**"ตาราง Payroll พร้อม Slip_PTC.xlsx"** — ใช้สูตรคำนวณเดียวกับไฟล์ Excel และเก็บข้อมูล
ที่กรอกไว้ใน Google Drive ของผู้ใช้ผ่าน Google Apps Script

เป็น static site (HTML + CSS + JavaScript ล้วน) ไม่มี backend ของตัวเอง จึงสามารถ
โฮสต์ฟรีบน GitHub Pages ได้ทันที

## โครงสร้างไฟล์

```
payroll-webapp/
├── index.html   หน้าเว็บหลัก (ตารางเงินเดือน + สลิป)
├── style.css    สไตล์ (สีอ้างอิงจากไฟล์ Excel ต้นฉบับ)
├── app.js       ตรรกะการคำนวณ, การแสดงผล, การ Export PDF, การเชื่อมต่อ Google Drive
├── Code.gs      สคริปต์ฝั่ง Google Apps Script (deploy แยกต่างหาก)
└── README.md    ไฟล์นี้
```

## สูตรคำนวณที่ใช้ (อ้างอิงจากไฟล์ Excel)

| รายการ | สูตร |
|---|---|
| รวมเงินได้ (M) | เงินเดือน + ค่าตำแหน่ง + ค่าล่วงเวลา + คอมมิชชั่น + โบนัส + เงินได้อื่นๆ |
| ประกันสังคม (N) | `IF(เงินเดือน<=1650, 1650×rate%, IF(เงินเดือน<=15000, เงินเดือน×rate%, 15000×rate%))` (ฐาน 1,650–15,000 ตามเกณฑ์ สปส., อัตราเริ่มต้น 5% ต่อคน ปรับได้รายคน) |
| รวมเงินหัก (R) | ประกันสังคม + ภาษีเงินได้ + สำรองเลี้ยงชีพ + หักเงินกู้ยืม + หัก ณ ที่จ่าย 3% |
| รวมเงินได้สุทธิ (S) | รวมเงินได้ − รวมเงินหัก |

> **หมายเหตุการแก้ไข:** ไฟล์ต้นฉบับมีจุดที่สูตรน่าจะพลาด 2 จุด ซึ่งแก้ไขแล้วในเว็บนี้:
> 1. สูตร "รวมเงินได้" เดิม `=SUM(G:K)` ไม่ได้รวมคอลัมน์ "เงินได้อื่นๆ" (L) จึงแก้ให้รวมด้วย
> 2. สูตรในชีต Slip ช่อง "เงินอื่นๆ" เดิมติดลบ (`=-INDEX(...)`) แก้เป็นค่าบวกตามปกติ
>
> ภาษีเงินได้ (O), หักเงินกู้ยืม (P) และหัก ณ ที่จ่าย 3% (Q) เป็นค่าที่กรอกเอง เช่นเดียวกับ
> ไฟล์ต้นฉบับ (มีปุ่ม "3%" ช่วยคำนวณ 3% ของรวมเงินได้ให้อัตโนมัติ)

## วิธีใช้งาน

1. เปิดหน้าเว็บ (ไฟล์ index.html หรือผ่าน GitHub Pages)
2. เลือก "งวดจ่ายเงินเดือน" แล้วกรอก/แก้ไขข้อมูลพนักงานและเงินได้-เงินหักในตาราง — ระบบคำนวณผลรวมให้อัตโนมัติ
3. ไปแท็บ "🧾 สลิปเงินเดือน" เลือกพนักงาน แล้วกด "ดาวน์โหลด PDF" เพื่อออกสลิปรายบุคคล
   (หรือกด "ส่งออกสลิปทุกคน (PDF)" ที่แท็บตารางเพื่อดาวน์โหลดทีละคนทั้งหมดในครั้งเดียว)
4. กด "☁ บันทึกลง Google Drive" เพื่อเก็บข้อมูลที่กรอกไว้ถาวร (ต้องตั้งค่าเชื่อมต่อก่อน ดูด้านล่าง)

ข้อมูลที่กรอกจะถูกบันทึกอัตโนมัติไว้ใน localStorage ของเบราว์เซอร์ด้วย (กันข้อมูลหายระหว่างใช้งาน)
แต่ localStorage ผูกกับเครื่อง/เบราว์เซอร์เดียว — ควรกด "บันทึกลง Google Drive" เพื่อให้เข้าถึงจากที่อื่นได้

## การเชื่อมต่อ Google Drive (Google Apps Script) — ทำครั้งเดียว

เนื่องจากเว็บนี้เป็น static site ไม่มีเซิร์ฟเวอร์ของตัวเอง การบันทึกข้อมูลลง Google Drive
จึงทำผ่าน **Google Apps Script Web App** ซึ่งจะสร้างและอัปเดต Google Sheet ชื่อ
"PTC Payroll Data" ไว้ใน Google Drive ของบัญชีที่ deploy ให้อัตโนมัติ (ผู้ใช้เว็บไม่ต้อง login Google)

ขั้นตอน:

1. ไปที่ https://script.google.com/ แล้วสร้างโปรเจกต์ใหม่
2. ลบโค้ดเริ่มต้น แล้ววางเนื้อหาทั้งหมดจากไฟล์ `Code.gs` แทน
3. กด **Deploy → New deployment**
   - Select type: **Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
4. กด Deploy แล้วอนุญาตสิทธิ์ตามที่ Google ถาม
5. คัดลอก **Web app URL** (ลงท้ายด้วย `/exec`)
6. กลับมาที่หน้าเว็บ Payroll → กด **⚙ ตั้งค่า** → วาง URL → บันทึก
7. กด **☁ บันทึกลง Google Drive** เพื่อทดสอบ — ควรมี Google Sheet ชื่อ "PTC Payroll Data" ปรากฏใน Drive ของบัญชีที่ deploy

หากต้องการแก้ไข/อัปเดตสคริปต์ภายหลัง ให้แก้ใน script.google.com แล้วกด Deploy → Manage deployments →
แก้ไข deployment เดิม (เลือกเวอร์ชันใหม่) เพื่อให้ URL เดิมยังใช้ได้

## การนำขึ้น GitHub Pages

โปรเจกต์นี้เป็น HTML/CSS/JS ล้วน สามารถอัปโหลดไฟล์ในโฟลเดอร์นี้ทั้งหมดขึ้น GitHub repository
(public) แล้วเปิดใช้งาน GitHub Pages จาก Settings → Pages → Branch: main / (root) ได้ทันที
โดยไม่ต้อง build ใดๆ

## ข้อจำกัดที่ควรทราบ

- ภาษีเงินได้บุคคลธรรมดาต้องกรอกยอดเอง (ไฟล์ต้นฉบับก็ไม่มีสูตรคำนวณภาษีอัตโนมัติ)
- Google Apps Script Web App มีข้อจำกัดโควตาการเรียกใช้งานของ Google (ปกติเพียงพอสำหรับบริษัทขนาดเล็ก-กลาง)
- ข้อมูลบันทึกแบบ "เขียนทับทั้งหมด" ทุกครั้งที่กดบันทึก (sync ทั้งชุดข้อมูล) ไม่ใช่แบบเพิ่มทีละแถว
